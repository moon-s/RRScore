# Manuscript Pipeline Run Order

This guide documents the expected run order after reorganization. It is not a request to recompute analyses.

## Critical Rule

Do not change scientific logic, thresholds, statistical methods, filtering rules, model parameters, random seeds, figure values, or output content during code-cleaning passes.

## 01 Processing Datasets

Directory: `01_processing_datasets`

Purpose: prepare DHS-overlapping SNPs, QTL/GWAS inputs, LD clumping/pruning, FinnGen RLS SNVs, and integrated datasets.

Runtime/data: large GWAS/QTL summary statistics and LD references required; potentially long runtime.

## 02 Processing Borzoi

Directory: `02_processing_borzoi`

Purpose: parse GTF annotations, prepare variants, map SNPs to genes, run Borzoi expression-delta scoring, select candidate variant-gene pairs, and link regulatory variants to GWAS.

Runtime/data: large genome/model files and Borzoi inference required; long runtime.

## 03 MR Analysis

Directory: `03_mr_analysis`

Purpose: run three-method MR analyses for RLS brain, blood, and single-cell QTL instruments; generate supporting MR summaries/plots.

Runtime/data: MR-ready instruments, harmonized GWAS/QTL files, and R MR packages required.

## 04 Network Model

Directory: `04_network_model`

Purpose: build graph datasets, run RWR/network analyses, summarize model performance, and produce calibrated brain model support tiers.

Runtime/data: upstream MR/Borzoi inputs and graph files required.

## 05 GSVA Analysis

Directory: `05_gsva_analysis`

Purpose: run bulk and single-cell GSVA/R-star analyses, including Figure 5 pseudobulk and single-cell stages.

Runtime/data: large expression matrices and GSVA dependencies required; long runtime for single-cell stages.

## 06 Main Figures

Directory: `06_main_figures`

Purpose: render final manuscript figure panels from finalized upstream outputs.

Runtime/data: expects completed upstream outputs; should not recompute heavy upstream analyses.

## 07 Tables

Directory: `07_tables`

Purpose: collect, index, or validate manuscript and supplementary tables. Some table outputs are currently generated by upstream scripts and documented in `documentation/`.

## Validation

Run `python smoke_test_pipeline.py` from the publication-ready root to check expected paths, dependencies, schemas, and final outputs without reanalysis.

## Publication-Ready Run Commands

These commands document script entry points after reorganization. Heavy analyses should be run only when required inputs are available. Python scripts modified in pass 6 accept optional `--config 00_config/paths.yaml` metadata while preserving existing defaults.
